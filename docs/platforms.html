Auto-detection: You don't need to specify your platform. setup.sh detects it automatically using /etc/os-release, uname, and platform-specific environment variables.

How Platform Detection Works

The detection logic in scripts/platform-detect.sh checks in this order:

  1. macOS — uname == "Darwin"
  2. FreeBSD — uname == "FreeBSD"
  3. WSL2 — kernel string contains "microsoft"
  4. Windows — uname -s starts with MINGW, MSYS, or CYGWIN
  5. Termux — ANDROID_ROOT or Termux PREFIX set
  6. ChromeOS — /dev/.cros_milestone exists
  7. Linux distros — ID field from /etc/os-release

Ubuntu derivatives (Linux Mint, Pop!_OS) are detected as Ubuntu. Arch derivatives (Manjaro, EndeavourOS, Garuda) are detected as Arch.

Linux

Fedora

Uses dnf. Straightforward — most packages are available in the default repos. EPEL-style repos may be needed for some tools on older releases.

Ubuntu / Debian

Uses apt. Some modern CLI tools (bat, fd-find, eza) may have different package names or require PPAs on older releases. The platform configs handle name mapping.

Arch Linux

Uses pacman. Excellent package coverage in official repos and the AUR. Arch derivatives (Manjaro, EndeavourOS, Garuda) are auto-detected and use the same config.

Gentoo

Uses emerge with binary packages (--getbinpkg) for faster installs. Package names use the full category/name format (e.g., app-editors/neovim). The platform-specific setup script configures binary package repositories.

Note: Gentoo installs take longer than binary distros even with binpkgs. The dry-run flag is useful to preview what will be compiled.

Void Linux

Uses xbps-install. Good package coverage. The platform setup script handles any Void-specific initialization.

Alpine Linux

Uses apk. Alpine's musl-based userland means some packages behave differently. The platform config maps to Alpine-specific package names where needed.

OpenSUSE

Uses zypper. Supports both Tumbleweed (rolling) and Leap (stable). Package names are generally consistent with Fedora.

Rocky Linux / AlmaLinux

Uses dnf. RHEL-compatible distros with similar repos. EPEL is typically needed for modern CLI tools. Both use the same package naming conventions as Fedora.

RaspberryPiOS

Uses apt (Debian-based). Defaults to the minimal profile since Raspberry Pis are often resource-constrained. ARM packages are generally available for all core tools.

NixOS

Uses nix for package management. machine-setup works on NixOS, but you may prefer the included Nix flake for a more declarative approach:

# Minimal dev shell
nix develop

# Full dev shell
nix develop .#full

macOS / FreeBSD

macOS

Uses Homebrew. If Homebrew isn't installed, setup.sh installs it automatically. Supports both Intel and Apple Silicon Macs. Homebrew cask is used for GUI apps when needed.

FreeBSD

Uses pkg with ports fallback. If a package isn't available as a binary, the installer falls back to building from the ports tree. The platform setup script ensures the ports tree is checked out.

Windows

Windows 11

Uses winget (Windows Package Manager). Run from Git Bash, MSYS2, or an elevated PowerShell. Some packages have different names in the winget repository — the platform config handles mapping.

WSL2

Uses apt (detected as a separate platform from native Ubuntu). WSL2 provides a full Linux environment, so most packages work identically to Ubuntu/Debian. Detection checks for "microsoft" in the kernel version string.

Mobile / Embedded

Termux (Android)

Uses pkg (Termux's package manager). Runs in a terminal emulator on Android. Some packages have Termux-specific names or aren't available. The platform config maps accordingly.

ChromeOS (Crostini)

Uses apt inside the Linux (Crostini) container. Detected via /dev/.cros_milestone. Behaves like Debian once inside the container.

Platform Package Configs

Each platform has a config file in packages/platforms/ that maps generic package names to platform-specific ones. For example:

# packages/platforms/gentoo.conf
[packages]
neovim = app-editors/neovim
ripgrep = sys-apps/ripgrep
fd-find = sys-apps/fd

# packages/platforms/ubuntu.conf
[packages]
fd-find = fd-find
bat = bat

If a package doesn't need remapping, the common name from the profile is used as-is.
