Installation

Get machine-setup running on your system in under a minute.

Prerequisites

machine-setup is pure bash and requires only standard system tools:

  • bash (4.0+ recommended, 3.2+ works on macOS)
  • git
  • curl (only for the one-liner install method)

Most systems have these pre-installed. If not, install git and curl with your system package manager first.

Quick Install (One-Liner)

The fastest way to get started. This downloads setup.sh and auto-clones the repo to ~/.machine-setup:

curl -fsSL https://raw.githubusercontent.com/gregnazario/machine-setup/main/setup.sh | bash

To specify a profile directly:

curl -fsSL https://raw.githubusercontent.com/gregnazario/machine-setup/main/setup.sh | bash -s -- --profile minimal

What this does: Clones the repository to ~/.machine-setup, detects your platform, installs packages, links dotfiles, and optionally sets up Syncthing and backups.

Manual Install (Clone)

Clone the repo and run setup yourself:

git clone https://github.com/gregnazario/machine-setup.git
cd machine-setup
./setup.sh

Or clone to a specific location:

git clone https://github.com/gregnazario/machine-setup.git ~/my-setup
cd ~/my-setup
./setup.sh --profile full

Choosing a Profile

Profiles control what gets installed and configured. Pick the one that matches your use case:

Profile Use Case What's Included
minimal Servers, Raspberry Pi, constrained systems nushell, neovim, ripgrep, fd, fzf, git, mise, syncthing
full Dev workstations Everything in minimal + zellij, bat, eza, dust, Docker, kubectl, Python, Rust, and more
selfhosted Home servers running Docker services Minimal + Docker Compose, Tailscale, monitoring tools, backup

See Configuration for community profiles (devops, creative, data-science, homelab) and creating your own.

# List all available profiles
./setup.sh --list-profiles

# See what a profile includes
./setup.sh --show-profile full

# Dry run — see what would happen without doing it
./setup.sh --dry-run --profile full

Skipping Steps

Don't need everything? Skip individual steps:

# Only install packages (no dotfiles, syncthing, or backup)
./setup.sh --profile full --no-dotfiles --no-syncthing --no-backup

# Only link dotfiles
./setup.sh --profile full --no-packages --no-syncthing --no-backup

Verify Installation

After setup completes, verify everything is working:

# Run a health check
./setup.sh --check --profile full

# View status dashboard
./setup.sh --status

# Detect any broken symlinks or conflicts
./setup.sh --detect-conflicts

Updating

Pull the latest changes and re-run setup:

./setup.sh --update

This runs git pull and then re-runs the setup with your current profile.

Uninstalling

Remove dotfile symlinks managed by machine-setup:

./setup.sh --unlink --profile full

Packages installed by your system package manager are not removed automatically. Use your package manager directly if you want to remove them.

Remote Setup via SSH

Set up a remote machine without logging in:

./scripts/remote-setup.sh user@host --profile minimal

This copies the repo to the remote machine and runs setup over SSH.

Interactive Setup

Not sure what you need? Run the interactive wizard:

./setup.sh --interactive

The wizard walks you through platform detection, profile selection, and optional features step by step.

Platform-Specific Notes

  • macOS: Homebrew is installed automatically if missing.
  • Gentoo: Uses binary packages (emerge --getbinpkg) for faster installs.
  • FreeBSD: Uses pkg with ports fallback for packages not in the repo.
  • NixOS: Consider using nix develop with the included flake instead of setup.sh for packages.
  • Windows 11: Uses winget. Run from an elevated PowerShell or Git Bash.
  • Termux: Uses pkg. Some packages have different names.

See Platforms for the full breakdown.
